1 Bidder Payment Submission

When bidders want to participate in reserve auctions, they must:

  • View the payment modal with QR code and bank details
  • Make payment via UPI or bank transfer
  • Upload payment screenshot as verification
  • Submit payment request for admin approval
POST /api/auction/payment-verification

2 Admin Review Interface

Admin can access the payment verification interface with:

  • Three-tab interface: Auctions, Requests, Payments
  • Comprehensive payment requests table
  • User details and auction information
  • Payment screenshot viewing capability
  • Approval and rejection actions
GET /api/admin/payments/payment-requests

3 Payment Approval Process

Admin can approve or reject payment requests:

  • Approve payments with optional admin notes
  • Reject payments with mandatory rejection reason
  • Real-time status updates
  • Automatic bidding eligibility activation
POST /api/admin/payments/payment-requests/:id/approve
POST /api/admin/payments/payment-requests/:id/reject

4 Bidding Access Control

Only approved bidders can participate in auctions:

  • Payment verification check before bid placement
  • Conditional bidding based on approval status
  • Real-time bidding eligibility validation
  • Secure auction participation workflow
POST /api/auction/bid (with payment verification)
